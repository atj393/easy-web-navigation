# Support

Thanks for using Easy Web Navigation. Here is how to get help.

## Bugs and feature requests

Open a **GitHub issue** using the provided templates:

- **Bug report** — something isn't working as expected.
- **Feature request** — an idea or improvement.
- **Accessibility rule** — propose a new or improved scanner rule.

Issues:
https://github.com/atj393/easy-web-navigation/issues

## Security vulnerabilities

Do **not** open a public issue for an exploitable vulnerability. Follow the
[Security Policy](SECURITY.md) to report it privately.

## Discussions

GitHub Discussions may be enabled later. Until then, please use issues.

## What support does NOT cover

- **No legal or accessibility-compliance certification support.** Easy Web Navigation is an
  inspection/assistive tool.
- **No guarantee that runtime inspection equals WCAG compliance.** A clean report is not a
  compliance pass; full accessibility requires source-level remediation, manual testing, and user
  testing with assistive technologies.

## Details to include in a bug report

To help us reproduce and fix issues quickly, please include:

- **Browser** and version (Chrome / Edge / Firefox).
- **Extension build** (development via `pnpm dev`, or a packaged build).
- **Page type** (e.g. a public URL, or one of the bundled demo pages).
- **Steps to reproduce.**
- **Expected behavior.**
- **Actual behavior.**
- **Screenshots** (only if they contain no sensitive information).
- Whether it also happens on the bundled **demo pages**.
- Any **console errors**.
