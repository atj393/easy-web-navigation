TAB-ORDER ALGORITHM (packages/keyboard-engine/src/index.ts)
  computeTabPath(doc, options) -> { items, summary, elements }
    1. createScanContext + collectFocusableElements (reused from dom-scanner;
       already excludes negative tabindex, hidden, disabled; open-shadow aware).
    2. sortFocusableElementsForTabOrder: positive tabindex first (ascending,
       ties keep DOM order via stable index), then tabindex 0 / natural in DOM
       order. Negatives dropped.
    3. Cap to options.maxItems ?? DEFAULT_TAB_PATH_CAP (100). summary =
       { shown, totalDetected, capped }.
    4. items: { index(1-based), selector, elementPreview, tagName,
       accessibleName?, tabIndex, rect? }.
  Also exports: sortFocusableElementsForTabOrder, createTabPathSummary,
  resolveTabIndex, DEFAULT_TAB_PATH_CAP.

OVERLAY MARKER BEHAVIOR (packages/focus-overlay/src/index.ts)
  New: showTabPath(elements), clearTabPath(), getTabPathState(), hasContent().
  - Numbered badge (.tab-num) + dashed outline (.tab-box) per element, in the
    same extension-owned shadow-isolated, pointer-events:none, fixed container.
  - positionMarker uses getBoundingClientRect; reposition() handles both
    highlights and tab markers on scroll/resize (rAF-throttled); disconnected
    markers are dropped.
  - clearTabPath keeps focus/locate highlights; unmount/destroy clears markers
    + highlights + container.
  - NEVER mutates inspected nodes.

CONTENT SCRIPT (apps/extension/entrypoints/content.ts)

  - TOGGLE_TAB_PATH: enable -> computeTabPath(document, options) +

  - GET_TAB_PATH_STATE -> TAB_PATH_RESULT { enabled, summary }.

IMPORTANT DECISIONS
  - keyboard-engine reuses dom-scanner's focusable detection (one-directional
    dep keyboard-engine -> dom-scanner -> wcag-rules -> shared-types; no cycle).
  - computeTabPath returns `elements` (live refs) alongside serializable items
    so the content script can drive the overlay without recomputing; only the
    summary is sent to the popup.
  - Default cap 100 (DEFAULT_TAB_PATH_CAP) to avoid rendering hundreds of
    markers; popup shows shown/total + capped warning.
  - Overlay stays mounted while focus helper OR tab path needs it; disabling
    one does not tear down the other.
  - getBoundingClientRect mocked in tests (jsdom has no layout).
  - No MutationObserver: tab path is a snapshot; toggle off/on to recompute.
  - missing-visible-focus still deferred (no reliable conservative heuristic).

KNOWN LIMITATIONS
  - Tab path is computed (DOM order + tabindex), not observed by pressing Tab;
    edge cases (inert, custom widgets, scroll containers) may differ from real
    browser/AT behavior.
  - Snapshot in time; large pages capped (default 100).
  - Closed shadow DOM and cross-origin iframes excluded; open shadow order is
    approximate.
  - Markers are visual only and do not change tab order.
  - Restricted pages (chrome:// etc.) -> friendly failure.
  - Firefox build is MV2 (WXT default); Chrome/Edge MV3. No icons yet.

WHAT WAS INTENTIONALLY NOT IMPLEMENTED
  - No assist mode. No changing tab order, focus(), or keyboard behavior.
  - No auto-fix, no inspected-node mutation, no ARIA injection, no added
    tabindex/handlers/skip links.
  - No auto-scroll (tab path does not scroll; locate still scrolls only on the
    explicit user click).
  - No MutationObserver / live re-render (manual re-toggle instead).
  - No AI, no speech, no broad host permissions, no remote, no publish, no PR.

  2. Serve demos: pnpm -C apps/demo-sites preview (http://localhost:4321).
  3. Open semantic-good-page.html, click "Show tab path"; numbered markers
     follow expected keyboard order.
  4. Open broken-keyboard-page.html; positive-tabindex items (1,2,3) appear
     first in the numbering.
  5. Open fake-buttons-page.html; non-focusable fake buttons are NOT numbered.
  6. Click "Hide tab path"; markers disappear.
  7. Turn on focus helper and Tab through; focus rectangle still works
     alongside / after tab path.
  8. Run a scan and "Locate on page"; Phase 0B/0C still work.
  9. Open a chrome:// page; friendly failure, no crash.
  10. Export report; no compliance claim.
