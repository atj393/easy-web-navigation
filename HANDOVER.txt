ROOT CAUSE / PRODUCT DECISION FOR THE "THIS PAGE ONLY" MESSAGE
  "This page only" relies on activeTab, which the browser grants for the page
  you act on. The background cannot inject into pages you navigate to in this
  scope, so automatic checking does not silently continue across navigations;
  it re-applies when you reopen the extension on a supported page. This is
  browser permission behavior and is NOT bypassed and adds NO permissions.
  Product decision: make the limitation unmistakable in plain English, shown
  directly under the status line both before and after checking starts:
    "This choice checks the page you started on. When you open another page,
     you may need to open Easy Web Navigation again."
  Plus, before starting, a guidance line:
    "To keep checking pages as you browse, choose This website or All websites
     before starting automatic checking."
  "This website" / "All websites" explanations use "browser access" wording.
  No technical terms (activeTab, host permission, cross-origin, SPA, route,
  injection, browser API) appear in the popup.

KEYBOARD-PATH MARKER-LIMIT BEHAVIOR
  New persisted preference tabPathMaxItems: 100 | 250 | 500 (default + capped
  at 500). Stored in local:monitoring alongside the other remembered prefs so
  it also applies during automatic checking. Old stored settings without the
  key fall back to 100 via normalizeTabPathMaxItems (existing users unaffected).
  - Popup control "Number of keyboard path markers" with options
    "100 markers (recommended) / 250 markers / 500 markers" and helper text
    "More markers can make very large pages slower and harder to read." It sits
    directly under the Keyboard path control and stays available even before the
    path is shown.
  - Changing the limit saves immediately and, if the path is visible, redraws it
    on the current page right away (no toggle off/on needed).
  - The limit is threaded popup -> settings -> APPLY_MONITORING / TOGGLE_TAB_PATH
    -> content script -> computeTabPath(maxItems) -> overlay. It is used for
    manual activation, automatic-checking re-application after navigation, SPA
    route refresh, and popup-open reapply.
  - Summary wording uses "keyboard items" (never "focusable items") and the real
    detected total: "Showing all 78 keyboard items." when nothing is limited, or
    "Showing the first 100 of 342 keyboard items." + "Choose a higher limit to
    show more." when limited. totalDetected is always the count BEFORE the limit.

EXPLICIT BEHAVIOR NOTES
  - "This page only" may require opening the extension again after navigating to
    another page. This is intended browser-permission behavior and is stated in
    the popup; it is not a bug and is not bypassed.

    There is no free numeric input and no value above 500.
  - No scanner rules, monitoring permission model, monitoring behavior, overlay
    ownership, or extension purpose changed. The only new behavior is applying
    the user-selected marker limit. Still read-only; no page-node mutation
    outside the extension-owned overlay; no ARIA injection; no tab-order change.
