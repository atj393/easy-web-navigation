IMPORTANT DECISIONS
  - Rule logic lives in wcag-rules but is DOM-agnostic: the scanner injects a
    RuleContext (query/isVisible/isFocusable/getAccessibleName/selector/preview).
    This keeps deps one-directional and avoids a cycle.
  - scanDocument signature changed to scanDocument(doc?, options?, now?) and
    returns the new ScanResult (no requestId/stats/placeholder). now() is
    injectable for deterministic tests.
  - A11yIssue.wcag replaces the old `criteria`; `message` replaced by
    title+description. canAutoEnhance always false in Phase 0B.
  - Tests run under jsdom (added as devDependency). jsdom has no layout, so
    visibility checks rely on attributes + computed display/visibility only.
  - browser.scripting.executeScript files must be root-absolute
    ("/content-scripts/content.js") per WXT's ScriptPublicPath type.
  - missing-skip-link only fires when nav/header/banner exists, to avoid noise.

KNOWN LIMITATIONS
  - DOM-observable only: JS click handlers added via addEventListener are not
    detected (some script-only "fake buttons" won't be flagged).
  - Accessible-name computation is a pragmatic subset of the ARIA accname algo.
  - Visibility detection is best-effort without layout.
  - Open shadow DOM only; closed shadow roots and cross-origin iframes skipped.
  - Firefox build is MV2 (WXT default); Chrome/Edge is MV3.
  - No extension icons yet.
  - A clean scan is NOT a compliance pass.

WHAT WAS INTENTIONALLY NOT IMPLEMENTED
  - No assist mode, no focus overlay, no tab-path visualization.
  - No auto-fix, no DOM mutation, no ARIA injection, no added tabindex/handlers.
  - No AI, no speech.
  - No missing-visible-focus heuristic (deferred to avoid unreliable results).
  - No broad host permissions; no remote; no publish; no PR.

RECOMMENDED MANUAL BROWSER WALKTHROUGH
  1. pnpm dev  (loads chrome-mv3 dev build via WXT).
  2. Serve demos: pnpm -C apps/demo-sites preview  (http://localhost:4321).
  3. Open broken-keyboard-page.html, click the toolbar icon, "Scan current page".
     Expect positive-tabindex + (no main) missing-main-landmark, etc.
  4. Open fake-buttons-page.html -> expect clickable-not-focusable + unlabeled.
  5. Open form-labels-page.html -> expect unlabeled-form-input for the bad inputs.
  6. Open semantic-good-page.html -> expect few/no issues.
  7. Click "Export report" -> downloads keywise-report.md.
  8. Confirm restricted pages (chrome://) show a friendly error, not a crash.
