IMPORTANT DECISIONS
  - WXT pinned to ^0.20.0 and @wxt-dev/module-react pinned to EXACTLY 1.1.4.
    Reason: module-react >=1.1.5 pulls @vitejs/plugin-react v5/v6 which needs
    Vite 7, but WXT resolves Vite 6 here -> "./internal not defined" crash.
    1.1.4 uses plugin-react v4 (Vite 6 compatible). If bumping WXT later,
    re-check this pairing.
  - Permissions: activeTab, scripting, storage ONLY. NO host_permissions.
    The content script uses registration:"runtime"; WXT would normally add
    host_permissions:["<all_urls>"]. A `build:manifestGenerated` hook in
    wxt.config.ts deletes host_permissions so no broad host access is
    requested. Content script file is still built for on-demand activeTab
    injection in later phases.
  - Branch renamed from master to main.
  - Root "ci" script must be run as `pnpm run ci` (pnpm reserves `pnpm ci`).
  - Packages export raw TypeScript via package.json "exports"; no per-package
    build step. Vite (via WXT) and Vitest compile sources directly.

KNOWN LIMITATIONS
  - Firefox build is Manifest V2 (WXT default for Firefox). Chrome/Edge is
    MV3. Revisit if Firefox MV3 is required.
  - No extension icons yet (WXT builds without them; add in a later phase).
  - No real analysis: scanDocument returns a placeholder ScanResult and all
    rules are status=not-implemented.
  - pnpm warns about 2 deprecated transitive deps (glob@10.5.0, uuid@8.3.2);
    harmless for Phase 0A.

WHAT WAS INTENTIONALLY NOT IMPLEMENTED
  - No AI. No speech. No automatic DOM mutation / auto-fix / ARIA injection.
  - No legal compliance claims (WCAG/BITV/EN 301 549/EAA/ADA/Section 508).
  - No broad host permissions, no <all_urls> as a required permission.
  - No remote, no publish, no PR. No icons. No real rule logic. No overlays.
