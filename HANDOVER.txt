PERMISSION CHANGES
  Added optional_host_permissions: ["http://*/*","https://*/*"] (MV3, Chrome).
  Required permissions UNCHANGED: activeTab, scripting, storage.
  No required host_permissions. No "tabs" permission. No webRequest/cookies/etc.
  Firefox MV2 output does NOT carry optional host permissions (WXT drops them),
  so site/all-sites scopes fall back to current-tab on Firefox.

SUMMARY OF WHAT CHANGED
  - Added explicit, user-started monitoring mode (Start/Stop) with a scope
    selector: current-tab (default), site, all-sites. Status label in popup.
  - Start: persist settings, request optional host permission if the scope needs
    one (from the click gesture, using a cached active-tab URL), scan now, and
    re-apply remembered helpers (focus/tab path) via APPLY_MONITORING. Denied
    permission -> fall back to current-tab with a friendly message.
  - Stop: persist enabled:false, clear overlays via APPLY_MONITORING{false,false}.
  - Helper preferences (focusHelperEnabled/tabPathEnabled) persist when toggled.
  - Background: event-driven tabs.onUpdated auto-injects the read-only content
    script into navigated tabs ONLY for site/all-sites scope and ONLY where a
    host permission is granted (permissions.contains) and the URL is supported.
    No polling, no tabs permission, no inactive-tab scans.
  - Content script: on load reads local:monitoring; if enabled, re-applies
    helpers + runs one read-only scan. New APPLY_MONITORING message.
  - Pure helpers (apps/extension/lib/monitoring.ts): isSupportedPageUrl,
    originPatternFromUrl, hostPermissionsForScope, scopeNeedsPermission,
    scopeLabel — unit tested.
  - Options page: read-only "Monitoring & privacy" section.
  - Docs (README, architecture, limitations, roadmap), CHANGELOG, HANDOVER.

IMPORTANT DECISIONS
  - No required broad host permissions; site/all-sites use optional host
    permissions requested only on user action. Denial -> graceful current-tab
    fallback. Honest: current-tab cannot auto-rescan across navigations alone.
  - Avoided the "tabs" permission: background reads tab.url only for tabs the
    granted host permission already exposes; URL filtering via isSupportedPageUrl
    + permissions.contains.
  - Browser-API code kept thin and untested; pure helpers isolated + unit tested.
  - Permission request issued before other awaits (cached active URL) to stay
    within the click user-gesture in Chrome.
  - Firefox MV2 lacks optional host permissions -> documented fallback rather
    than per-browser manifest forking.

KNOWN LIMITATIONS
  - Current-tab monitoring is session/gesture limited (activeTab).
  - Site/all-sites require user-granted optional host permissions; unavailable on
    Firefox MV2 (falls back to current-tab).
  - Restricted browser pages cannot be scanned (friendly failure).
  - SPA in-page route changes are not auto-rescanned yet (no MutationObserver).
  - Background auto-injection is event-driven on navigation; no inactive-tab
    scanning.
  - Local folder still keywise-web; missing-visible-focus still deferred.

WHAT WAS INTENTIONALLY NOT IMPLEMENTED
  - No always-on/hidden automation; monitoring is explicit and user-controlled.
  - No tabs/webRequest/cookies/history/downloads/clipboardRead permissions.
  - No analytics, no external/AI calls, no speech, no auto-fix, no ARIA
    injection, no tab-order change, no page mutation (overlay only).
  - No SPA MutationObserver, no store publishing.
