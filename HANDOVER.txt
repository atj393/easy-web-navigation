REPORT GENERATOR (packages/report-generator/src/index.ts)
  Exports: generateMarkdownReport(result, options?), generateJsonReport(result,
  options?), DISCLAIMER, PRODUCT_NAME, REPORT_TYPE, SCHEMA_VERSION.
  options: ReportOptions { tabPathSummary?: TabPathSummary }.
  Pure / DOM-independent -> easy to unit test.

CLIPBOARD HELPER (apps/extension/lib/clipboard.ts)
  copyTextToClipboard(text, env?: { clipboard?, doc? }) -> Promise<boolean>.
  Async API first, then textarea+execCommand fallback, else false. No auto-copy.

POPUP (apps/extension/entrypoints/popup/App.tsx)
  buildReport() -> generateMarkdownReport(result, { tabPathSummary }).
  copyReport() uses copyTextToClipboard; downloadReport() Blob+anchor.
  reportStatus messages: "Report copied to clipboard." / "Could not copy report.
  Download is still available." / "Run a scan before exporting a report." /
  "Report downloaded." Buttons disabled until a scan result exists.

IMPORTANT DECISIONS
  - Report generator stays a pure function; tab-path summary is passed in via
    ReportOptions rather than plumbed through ScanResult, keeping the scan model
    unchanged.
  - "Runtime-observable only" is shown as a fixed per-issue line (all current
    rules are DOM-observable); no new A11yIssue field was added.
  - Clipboard helper lives in apps/extension/lib and is dependency-injected so it
    is unit-testable without a real browser clipboard; vitest include broadened
    to apps/**/lib/**/*.test.ts.
  - JSON report omits tabPathSummary entirely when not provided (matches
    "tabPathSummary if provided").

KNOWN LIMITATIONS
  - A report reflects one scan; a clean report is not a compliance pass.
  - Clipboard API may be unavailable (older/insecure contexts) -> textarea
    fallback -> if that fails, user is told download is still available.
  - Tab-path summary in the report is a runtime visual aid, not an audit metric.
  - Local root folder still named keywise-web; Firefox build is MV2; no icons.
  - missing-visible-focus still deferred (unchanged by this phase).

WHAT WAS INTENTIONALLY NOT IMPLEMENTED
  - No new scanner rules, no overlay/tab-path logic changes (only report data
    plumbing).
  - No report preview UI, no advanced filters, no settings expansion.
  - No inspected-node mutation, no focus change, no tab-order change, no ARIA
    injection, no auto-fix.
  - No AI, no speech, no permission changes, no remote, no publish, no PR.

RECOMMENDED MANUAL BROWSER WALKTHROUGH
  1. pnpm dev (loads chrome-mv3 dev build).
  2. Serve demos: pnpm -C apps/demo-sites preview (http://localhost:4321).
  3. Open form-labels-page.html or fake-buttons-page.html; "Scan current page".
  4. "Copy Markdown report" -> paste into an editor; confirm clean sections +
     disclaimer.
  5. "Download Markdown report" -> open easy-web-navigation-report.md.
  6. Toggle "Show tab path", then copy/download again; confirm the report's tab
     path summary says "Runtime visual aid only - not an audit metric".
  7. Open a chrome:// page; confirm friendly failure remains.
  8. Confirm no compliance claim appears anywhere.
