ROOT CAUSE (user: "helpers don't auto re-apply; must click toggles again")
  1) Popup-open while monitoring ON only READ helper state (GET_* with no
     injection). In current-tab scope the content script isn't present on a new
     page, so nothing re-applied and the toggles showed off.
  2) Start wrote focusHelperEnabled/tabPathEnabled from popup UI state, and Stop
     reset those toggles to false -> a Stop->Start cycle lost the remembered
     preferences.
  3) SPA refreshForRoute used stale in-memory closure values.

FIX SUMMARY
  - Popup useEffect: when monitoring is enabled, inject into the active tab
    (popup open = user gesture), re-apply remembered helpers via APPLY_MONITORING
    (payload from saved settings) and scan. When off, best-effort GET_* (no
    inject). This makes helpers re-appear on each supported page, incl.
    current-tab when the popup is opened.
  - startMonitoring: persists only { enabled, scope } and PRESERVES saved helper
    prefs, then applies createApplyMonitoringPayload(savedSettings). stopMonitoring
    keeps prefs (only enabled:false) and clears overlays.
  - content refreshForRoute: now async; reads latest monitoringItem prefs on each
    SPA route change (no stale values); returns early if monitoring disabled.
  - Added tested pure helpers in lib/monitoring.ts: mergeMonitoringSettings,
    updateMonitoringHelperPreference, createApplyMonitoringPayload,
    shouldAutoApplyHelpers.
  - Clearer popup wording (remember choices; current-tab cross-origin note;
    site/all-sites auto-apply note).

IMPORTANT DECISIONS
  - Saved monitoring settings are the single source of truth for helper prefs;
    popup UI state no longer overwrites them on Start.
  - Popup-open auto-apply is the mechanism that makes current-tab monitoring
    re-apply across pages (background can't inject in that scope). This is honest
    and documented; for hands-free cross-page behavior use site/all-sites.
  - SPA refresh reads storage so prefs changed elsewhere are respected.
  - No behavior change to scanner/report/overlay beyond the apply flow.

KNOWN LIMITATIONS
  - Current-tab scope: helpers re-apply on popup open (gesture), not silently as
    you browse; for that, grant site/all-sites optional host permission.
  - Site/all-sites unavailable on Firefox MV2 (falls back to current-tab).
  - SPA refresh is URL-signal based and best-effort (apps that change content
    without changing the URL won't trigger it).
  - Restricted browser pages can't be monitored (friendly message).
  - Local folder still keywise-web; missing-visible-focus still deferred.
