IMPORTANT DECISIONS
  - SPA detection is URL-signal based (History API + popstate + hashchange) with
    a trailing debounce (400ms). No always-on MutationObserver (kept lightweight,
    avoids continuous scanning).
  - History wrapping is defensive: original always called through, double-wrap
    guarded via a marker, restored on stop, and an `active` flag makes any
    leftover wrapper a pass-through.
  - Monitor lifecycle is tied to local:monitoring (load + storage.watch), so it
    starts/stops exactly with the user's monitoring choice; cleanup-safe.
  - refreshForRoute tears down then re-applies helpers so the overlay re-mounts
    even when the SPA replaced <body>; tab path is recomputed for the new DOM.
  - Browser-coupled wiring kept thin; pure/injectable helper fully unit-tested
    (injectable history/target/getLocation).

KNOWN LIMITATIONS
  - SPA refresh is best-effort and URL-driven: apps that change content WITHOUT
    changing the URL may not trigger a refresh (use manual scan).
  - Not framework-lifecycle-aware; heavy async pages may still need a manual
    re-scan after the debounce window.
  - Closed shadow DOM / cross-origin iframes remain excluded.
  - Current-tab scope still session/gesture limited; site/all-sites need optional
    host permissions (unavailable on Firefox MV2 -> current-tab fallback).
  - Local folder still keywise-web; missing-visible-focus still deferred.

WHAT WAS INTENTIONALLY NOT IMPLEMENTED
  - No always-on / heavy MutationObserver scanner loop; no continuous scanning.
  - No new permissions, no tabs/webRequest/cookies/history/downloads/clipboard
    permissions.
  - No page mutation (overlay only), no tab-order change, no ARIA injection, no
    auto-fix, no AI, no speech, no analytics, no external calls, no compliance
    claims.
  - No store publishing.
